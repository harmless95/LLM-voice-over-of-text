## LLM Voice Over of Text (Twitch)

Проект озвучивает сообщения из чата Twitch и позволяет голосом управлять режимом перевода с русского на английский и обратно. Используются Vosk (STT), Silero TTS и модели перевода Helsinki-NLP.

### Возможности

- **Озвучка чата Twitch**: сообщения из выбранного канала преобразуются в речь.
- **Обработка ника и текста**:
  - транслитерация латиницы в кириллицу;
  - проговаривание чисел словами;
  - озвучивание смайлов как описаний;
  - перевод английского текста в русский для TTS.
- **Голосовые команды (STT)**:
  - `Бобр старт` – включить режим перевода;
  - `Бобр стоп` – выключить режим перевода;
  - `Бобр статус` – сообщить текущий статус.

### Требования

- Python `>=3.11, <3.15`
- Поддерживаемая ОС (проверено на Windows 10/11)
- Микрофон и устройство вывода звука
- (Опционально) GPU с CUDA 12.8 для ускорения моделей (`torch` ставится отдельной группой зависимостей)

### Установка

Рекомендуется использовать Poetry.

```bash
cd my_project
poetry install           # основные зависимости
poetry install --with dev  # добавить dev-зависимости (black, pytest)
```

Если нужна поддержка CUDA для PyTorch, Poetry возьмёт колёса из дополнительного источника, указанного в `pyproject.toml`.

### Настройка `.env`

В директории `my_project/core` есть файл `.env.template`. Скопируйте его в `.env` и заполните значения:

- `APP_CONFIG__CONF_TW__TOKEN` – OAuth‑токен Twitch (без `oauth:` префикса, он добавляется в коде);
- `APP_CONFIG__CONF_TW__USERNAME` – имя бота/аккаунта;
- `APP_CONFIG__CONF_TW__CHANNEL` – канал без `#`;
- при необходимости можно изменить параметры логирования и путей к моделям (см. `core/config.py`).

Пример (упрощённо):

```env
APP_CONFIG__CONF_TW__TOKEN=your_twitch_oauth_token
APP_CONFIG__CONF_TW__USERNAME=your_bot_name
APP_CONFIG__CONF_TW__CHANNEL=your_channel_name
```

### Загрузка и размещение моделей

В конфиге (`core/config.py`) используются следующие пути по умолчанию (относительно `my_project`):

- Vosk STT: `data_models/vosk/vosk-model-small-ru-0.22`
- Модели перевода: `data_models/helsinki_nlp/` и `data_models/helsinki_ru_nlp/`

Убедитесь, что модели загружены в эти директории (либо измените пути в `ConfigSTT` / `.env`).

### Запуск бота

Из корня `my_project`:

```bash
poetry run python main_tw.py
```

После запуска:

- бот подключится к указанному каналу Twitch;
- сообщения из чата начнут озвучиваться;
- голосовые команды с микрофона будут переключать режим перевода.

### Тесты

В проекте есть базовые тесты для текстовой логики и команд:

- `tests/test_translate_text.py` – проверяет `main_message` (обработка ника и сообщения);
- `tests/test_commands.py` – проверяет класс `Commands` и поведение очереди.

Запуск тестов:

```bash
cd my_project
poetry run pytest
```

### Структура проекта (кратко)

- `my_project/main_tw.py` – точка входа, подключение к Twitch, запуск TTS/STT и обработка сообщений.
- `my_project/core/config.py` – конфиг (Pydantic Settings), логгер.
- `my_project/core/model/` – модули для STT, TTS, перевода и голосовых команд.
- `my_project/utils/translate_text.py` – обработка ника, текста, чисел и смайлов.
- `my_project/core/constants.py` – ключевые строковые и числовые константы.
- `my_project/tests/` – базовые юнит‑тесты.
